Note

This documentation can also be obtained by running housing-prices --help from the terminal window.

Or, to get help for a specific subcommand, run housing-prices <subcommand> --help.

housing-prices

housing-prices let’s you explore the housing price data presented in Chapter 2 of the book Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow (3rd ed.).

The following subcommands are available:

  • create-test-set : Create a test set from the housing price data.

  • describe-column : Print information about a specific column in the housing price data.

  • download-data : Download the housing price data from the book’s web page.

  • info : Print information about the housing price data.

  • plot-histograms : Plot histograms of the housing price data.

  • stratify-column : Stratify the data in a column of the housing price data.

housing-prices [OPTIONS] COMMAND [ARGS]...

Options

-v, --verbose

Show verbose output

create-test-set

housing-prices create-test-set creates a test set from the housing price data. The default splitting --method is to use a CRC (Cyclic Redundancy Check) to split the data. Stratified splitting can also be used with the STRATIFIED method. It requires a column name to stratify the data on. It is assumed that the column has been prepared in advance with the stratify-column command. The RANDOM method is not implemented yet. Method names can be given in lowercase or uppercase. Short names are also accepted, e.g. RND for RANDOM, and STR for STRATIFIED. The test set ratio can be set with the --test-ratio option. The default ratio is 0.2. The test ratio is the ratio of the test set size to the full dataset size. If method is STRATIFIED, the --stratify-column option can be used to specify the column to stratify the data on. See housing-prices stratify-column

housing-prices create-test-set [OPTIONS]

Options

-m, --method <method>

The method to use for creating the test set. Valid methods are: CRC, RANDOM, STRATIFIED

-r, --test-ratio <test_ratio>

The ratio of the test set size to the full dataset size. Default is 0.2

-c, --stratify-column <stratify_column>

The column to stratify the data on. Column must be prepared in advance with the stratify-column command.

Options

-c, --column-name <column_name>

The column name to plot. If not given, all columns are plotted. A list of valid column names can be obtained with the housing-prices info command.

stratify-column

housing-prices stratify-column COLUMN stratifies the data in COLUMN of the housing.csv data file. Use the --bins option to specify the bin edges. The number of bins is one less than the number of bin edges. The bin edges should be given as a comma-separated list of numbers. For example, --bins=0,100000,200000,300000 will create 3 bins: [0, 100000), [100000, 200000), [200000, 300000). See housing-prices info for information about the housing price column names, and housing-prices describe-column for information about the minimum and maximum column values. The strafified data bins are labeled with consequtive integers starting from 1. Finally, the stratified data is plotted as a histogram.

housing-prices stratify-column [OPTIONS] COLUMN_NAME

Options

-b, --bins <bins>

Required Comma-separated list of bin edges. The number of bins is one less than the number of bin edges.

Arguments

COLUMN_NAME
